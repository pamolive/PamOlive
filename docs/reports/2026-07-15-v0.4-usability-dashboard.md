# PAM-olive 0.4 usability and operations report

Date: 2026-07-15

## Scope

This iteration improves the daily user experience without weakening the security model.
It adds a light interface, three selectable languages, dynamic vault forms, personal
password groups, secret editing, a Home navigation entry, operational administration
metrics, the official PAM-olive visual identity, and password-based SSH onboarding.

## SSH behavior

An SSH target can now use one of two explicit host identity policies:

- **Trust on first use**: the first successful connection records and audits the server
  host key. Every later connection must present that same key.
- **Strict pre-approval**: an administrator must import an approved host key before the
  session can start.

Both modes support password credentials. PAM-olive never requires a client private key
when the selected target credential is a username and password.

## User experience

- Dark, light, and operating-system themes are stored per account.
- English, French, and Spanish can be changed from the top navigation or account page.
- Personal secrets can be grouped and edited only by their owner.
- Vault fields react to the selected item type so unrelated inputs are hidden.
- The main navigation includes Home, Passwords, Targets, and Administration when the
  signed-in user has console access.
- PAM-olive uses the supplied green and monochrome logo assets.

## Administration dashboard

The administration overview refreshes every 15 seconds and presents:

- database and Redis cache health;
- connected web users;
- active SSH/RDP sessions;
- failed password sign-ins during the last 24 hours;
- pending approvals;
- audit event volume and recent failure details.

The dashboard endpoints remain protected by the existing administration capability.

## Deployment safeguards

The NAS deployment procedure creates a new timestamped PAM-olive backup first, runs the
complete Docker test profile, and updates only `/volume1/docker/pam-olive`. It does not
modify unrelated Docker projects, NAS users, or data.
